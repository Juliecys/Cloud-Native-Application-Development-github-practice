# Cloud-Native-Application-Development-github-practice

Repo 操作 (20)
- 創建一個 GitHub Repo
- 有準備 README.md
- 從網頁上可以看到除了 Main 外有額外兩個 Branch（hw1-p、hw1-f）

Issue (20)
- 創建任意 Issue
- 有 Issue Template

Pull Request(20)
- 以 hw1-p 此 branch 對 main 創立 Pull Request
- 以 hw1-f 此 branch 對 main 創建 Pull Request
- 上述任何一個 PR 內有留言互動，針對程式碼變動留言

GitHub Action(40)
- 點選 GitHub Action 內有任何 Action 運行過痕跡
- Action 內扣除預設，至少要有額外兩個步驟(Steps)
- hw1-p 的 PR 必須要有 GitHub Action 運行，要成功
- hw1-f 的 PR 必須要有 GitHub Action 運行，要失敗
